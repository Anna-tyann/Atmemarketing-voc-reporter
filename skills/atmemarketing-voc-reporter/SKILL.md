# Atmemarketing VOC 报告生成 Skill

## 概述

接收打标 JSON → 生成完整 HTML 交互式报告。单文件，双击浏览器打开。**品类通用**，适配任意产品打标数据。

## 执行流程

### Step 1: 获取打标数据

```python
import json
with open('产品名_打标完成版.json', 'r', encoding='utf-8') as f:
    tags_data = json.load(f)
```

### Step 2: 准备产品配置

```python
product_config = {
    'product_name': '产品名称',
    'tag_system': {
        "产品维度1": {"color": "#2563eb"},
        "产品维度2": {"color": "#7c3aed"},
        # ... 只需传入产品功能维度（10个），行为场景维度颜色引擎内置
    }
}
```

### Step 3: 生成报告

```python
import sys
sys.path.insert(0, 'voc_engine.py所在目录')
from voc_engine import build_report_html

report_html = build_report_html(tags_data, product_config)

with open('产品名_VOC分析报告.html', 'w', encoding='utf-8') as f:
    f.write(report_html)
```

### Step 4: 预览

```bash
open 产品名_VOC分析报告.html  # macOS
# 或双击 HTML 文件
```

## 营销战略方向总结（最后一章）

报告最后板块，**完全基于行为维度标签数据动态生成**，品类通用。不再硬编码任何特定品类的场景或人群。

### 🎬 社媒内容策略
- 基于 **🎯购买动因** Top L2 标签 → 确定内容角度（口碑推荐 / 自用升级 / 送礼等）
- 基于 **📢声音类型** 中「推荐分享」「感谢致意」的高赞评论 → 提取 UGC 素材
- 基于 **🧠心理类型** 中高情感密度标签 → 确定情感 hook

### 🤝 海外网红合作方向
- 基于 **📍使用地点** Top 场景标签 → 匹配场景类博主
- 基于 **💡用户兴趣/生活方式** Top 标签 → 匹配兴趣圈层 KOL/KOC
- 基于 **👤用户画像** → 匹配受众重合的红人
- 基于 **📢声音类型** 中高赞 quote → 提供 Brief 脚本素材

### 🌐 网站与落地页优化
- 基于 **📍使用地点** Top 场景 → 确定 Hero 区视觉
- 基于 **📢声音类型** 中「推荐分享」高赞评论 → 首页 quote 轮播
- 基于 **🏷️品牌/竞品对比** → 竞品对比模块
- 基于 **🔄复购/忠诚行为** → 社交证明数据

### 🎯 精准投放人群包
- 基于 **👤用户画像 × 🎯购买动因 × 💡用户兴趣** 三维交叉 → 定义投放人群
- 基于 **维度交叉分析** 数据 → 确定每组人群的差异化痛点 hook
- 基于 **🏷️品牌/竞品对比** → 竞品截流定向

## 维度交叉分析（板块11）

报告中新增的交叉分析矩阵，计算同一评论中两个维度标签的共现关系：

| 交叉组合 | 分析问题 | 数据来源 |
|---------|---------|---------|
| 👤 人群 × 痛点 | 哪类用户最集中地遇到什么问题 | 用户画像 × 产品维度负面标签 |
| 📍 使用地点 × 痛点 | 什么情况下问题最容易出现 | 使用地点 × 产品维度负面标签 |
| ⏰ 使用场景 × 痛点 | 什么时机问题最突出 | 使用场景 × 产品维度负面标签 |
| 🔧 功能对象 × 痛点 | 哪个产品环节的问题最集中 | 产品维度间交叉 |
| 📢 声音类型 × 痛点 | 推荐、吐槽、比较分别在说什么 | 声音类型 × 产品维度负面标签 |
| 🔮 未满足 × 💥 实际后果 | 用户如何应对，为什么仍然不够 | 未满足需求 × 实际后果 |

交叉结果以**可排序表格**展示，同时配以 **ECharts 热力图** 直观呈现，方便快速定位高价值洞察。

## 可视化设计规范

- **散点图**：气泡大小 = √提及次数 × 9（非线性缩放，避免高频标签气泡过大）；50% 满意度参考线
- **热力图**：颜色渐变从浅蓝到深蓝，展示共现密度
- **词云图**：圆形布局，字号 14-52px，随机旋转 ±30°
- **颜色方案**：正面 `#22c55e`｜负面 `#ef4444`｜Header 渐变 `#1e3a5f→#2563eb→#7c3aed`
- **痛点卡片**：红色左边框 + 浅红背景
- **优势卡片**：绿色左边框 + 浅绿背景
- **移动端**（<768px）：自动单列布局

### CDN 依赖

- ECharts 5.5.0：`cdn.jsdelivr.net/npm/echarts@5.5.0/dist/echarts.min.js`
- ECharts WordCloud 2.1.0：`cdn.jsdelivr.net/npm/echarts-wordcloud@2.1.0/dist/echarts-wordcloud.min.js`

## 注意事项

- 报告是单文件 HTML，ECharts 从 CDN 加载（需网络）
- 洞察质量 > 数量
- 所有痛点/优势必须附带原文 quote
- 界面中文，原文保留原始语言
- **英文引用需附中文翻译**：用户原声 quote 如为英文，需在下方附上中文翻译，格式为「原文 + 翻译」
- **报告品类通用**：所有营销战略方向、用户画像、场景描述均基于传入的标签数据动态生成，不硬编码任何品类专属内容

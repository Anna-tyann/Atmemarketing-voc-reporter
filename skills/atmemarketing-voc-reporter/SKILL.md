# Atmemarketing VOC 报告生成 Skill

## 概述

接收打标 JSON → 生成完整 HTML 交互式报告。单文件，双击浏览器打开。**品类通用**，适配任意产品打标数据。

## 核心特性（v5.0）

- ✅ **品类通用** — 适配冲牙器 / 电动自行车 / 猫抓板 / 手机壳等任意品类
- ✅ **数据驱动画像与场景** — 从真实 `👤用户画像`、`📍使用地点`、`⏰使用场景` 标签直接提取，杜绝跨品类硬编码
- ✅ **气泡自适应** — 散点图根据样本量自动调整气泡大小（大数据自动缩小，小数据自动放大）
- ✅ **维度交叉分析矩阵** — 5个矩阵 + 热力图（人群×痛点、场景×痛点、声音类型×痛点、未满足×实际后果）
- ✅ **高频关键词词云** — Top 80 词，圆形布局 ECharts WordCloud
- ✅ **营销战略方向总结** — 完全数据驱动，每个产品生成专属策略，不重复

## 气泡自适应算法

```
L2 散点图：maxMention >200→0.25x | >100→0.4x | >50→0.8x | 默认 1.5x（上限60px）
L1 散点图：maxMention >200→0.3x | >100→0.5x | 默认 0.8x（上限80px）
```

## 执行流程

### Step 1: 获取打标数据

```python
import json
with open('产品名_打标完成版.json', 'r', encoding='utf-8') as f:
    tags_data = json.load(f)
```

### Step 2: 准备产品配置

```python
product_config = {
    'product_name': '产品名称',
    'tag_system': {
        "产品维度1": {"color": "#2563eb"},
        # ... 只需传入产品功能维度（10个），行为场景维度颜色引擎内置
    }
}
```

### Step 3: 生成报告

```python
import sys
sys.path.insert(0, 'voc_engine.py所在目录')
from voc_engine import build_report_html

report_html = build_report_html(tags_data, product_config)

with open('产品名_VOC分析报告.html', 'w', encoding='utf-8') as f:
    f.write(report_html)
```

### Step 4: 预览

```bash
open 产品名_VOC分析报告.html  # macOS
```

## 可视化设计规范

- **散点图**：气泡大小自适应（大样本自动缩小）；50% 满意度参考线
- **热力图**：颜色渐变从浅蓝到深蓝，展示共现密度
- **词云图**：圆形布局，字号 14-52px，随机旋转 ±30°，彩色渐变
- **颜色方案**：正面 `#22c55e`｜负面 `#ef4444`｜Header 渐变 `#1e3a5f→#2563eb→#7c3aed`
- **痛点卡片**：红色左边框 + 浅红背景
- **优势卡片**：绿色左边框 + 浅绿背景
- **移动端**（<768px）：自动单列布局

### CDN 依赖

- ECharts 5.5.0：`cdn.jsdelivr.net/npm/echarts@5.5.0/dist/echarts.min.js`
- ECharts WordCloud 2.1.0：`cdn.jsdelivr.net/npm/echarts-wordcloud@2.1.0/dist/echarts-wordcloud.min.js`
- Google Fonts：Noto Sans SC
